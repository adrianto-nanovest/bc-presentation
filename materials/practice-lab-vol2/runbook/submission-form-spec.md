# Wrap-up Submission Form — Spec

> Committee builds the actual Google Form from this spec. Form opens at T+85 on each session day, closes after Day 1 / Day 2 ends. One submission per team. Share the form URL on-screen and in the Drive folder.

---

## Form fields

| # | Field name | Type | Required | Validation | Notes |
|---|---|---|---|---|---|
| 1 | Team name | Short text | Yes | Non-empty | Used as soft duplicate-check key; committee flags duplicate team names manually |
| 2 | Persona | Multiple choice (radio) | Yes | Day 1 (Section Head Analyst) / Day 2 (Team Leader Booster) | One option only; sets context for judging |
| 3 | Artifact shareable link | URL | Yes | Must start with `https://` | Stage 3c output — Claude Artifact shareable link |
| 4 | NotebookLM workbook share link | URL | Yes | Must start with `https://` | Stage 1 workbook — gives judges access to Report + Slide Deck + Video |
| 5 | HTML design system file | File upload | Yes | `.html` only · ≤ 10 MB | Stage 3b Claude Design output |
| 6 | Reflection: which handoff felt most useful and why? | Paragraph (long text) | Yes | Soft: min 1 sentence, max 2 sentences (guidance in field hint) | Handoffs are A (Stage 1 → 2), B (diagram → Artifact), C (slide template → design system) |
| 7 | Authored SKILL.md | File upload | Yes | `.md` only · ≤ 100 KB | Stage 4 take-home — the Dashboard Maker Skill the team authored via skill-creator |
| 8 | Skill name | Short text | Yes | Non-empty | Must match the `name:` field in the uploaded SKILL.md exactly |

---

## Field hints (show below each field in the form)

| # | Hint text |
|---|---|
| 1 | The team name you chose at the start of the session. |
| 2 | Select the persona your team used today. |
| 3 | In Claude, open your Artifact → Share → Copy link. Paste here. |
| 4 | In NotebookLM, open your notebook → Share → Copy link. Paste here. |
| 5 | The `.html` file you downloaded from Claude Design in Stage 3b. |
| 6 | One or two sentences only. Which of the three handoffs — downloading the NotebookLM Report, converting the diagram into an Artifact, or extracting the design system from the slide template — felt most useful? Why? |
| 7 | The `.md` file you downloaded after the skill-creator interview in Stage 4. |
| 8 | Open your SKILL.md and copy the value of the `name:` field from the YAML frontmatter. Paste exactly. |

---

## Form settings

| Setting | Value |
|---|---|
| Access | Anyone with the link — no Google sign-in required (committee laptops may use a shared account) |
| One response per team | Enforced by honor; committee flags duplicate Team name values in the response sheet |
| Email collection | Off — no personal data needed |
| Response confirmation | Show a confirmation message: *"Submitted. Your team's work is recorded. Well done."* |
| Response destination | Linked Google Sheet: `BCE Vol-2 — Lab Submissions` |

---

## Response sheet columns (auto-generated by Google Forms)

The linked Sheet will have one column per form field plus a Timestamp column. Committee should rename columns to match the field names above after first response arrives.

Recommended column order in Sheet: `Timestamp · Team name · Persona · Artifact link · NotebookLM link · Reflection · Skill name · (file links auto-appended by Drive for uploads)`

---

## Output handoff

After both session days close:

1. Committee exports the response Sheet to CSV.
2. CSV + each team's submitted Artifact link → forwarded to the judging panel.
3. Uploaded files (HTML design system + SKILL.md) are accessible via the linked Drive folder that Google Forms attaches automatically — share that folder with the judging panel alongside the CSV.
