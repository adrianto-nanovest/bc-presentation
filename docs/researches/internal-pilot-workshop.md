# Nanovest Pilot Workshop Content Mining for Berau Coal Energy (BCE)

## Executive Summary

The Nanovest Pilot Workshop deck ("AI Workshop 2025: From Confusion to Mastery") contains **highly reusable, production-grade content** across all 9 BCE topics. Key findings:

1. **Immediate High-Value Reuse (Verbatim):** AI evolution journey (slides 11-13), prompt engineering fundamentals (20-26), context engineering layers (28), RAG/MCP architectures (34-37), agent/agentic AI progression (41-45). These transcend industry and need minimal adaptation.

2. **Strong Adaptation Candidates:** Process automation (BPM/RPA/IPA slides 17-19) maps directly to mining ops (ore processing, environmental compliance workflows); best practices framework (27) applies universally; tools ecosystem (38-40) shows vendor landscape mining teams must navigate.

3. **Visual/Animation Patterns Worth Borrowing:** Circular journey maps (slide 9 "AI Mastery Journey"), stage progression visualizations (Reactive → Single Agents → Agentic Systems, slide 41), before/after transformation diagrams, Venn diagram overlaps (context confusion, slide 32), layered context memory models (28-29).

4. **Critical Gap:** No mining-specific case studies or process examples. Nanovest examples (legal contracts, bank reconciliation, HR onboarding, customer service) must be **recontextualized to mining workflows:** equipment maintenance tickets, ore grade prediction, safety incident reporting, equipment failure forecasting.

5. **Operational Note:** Deck targets upskilling mid-career professionals in finance/tech; BCE audience is mining engineers + general workers. Tone/terminology consistent, but **anchor all technical concepts to tangible mining outputs** (production gains, safety reduction, cost per tonne).

## Visual/Animation Patterns Worth Borrowing

### 1. **Circular Journey Maps** (Slide 9)
- **Pattern:** Circular path with 5 location markers, each labeled with a topic + duration. Dashed path segments suggest progression but circularity implies "continuous learning loop."
- **Why it works:** Psychological anchor that learning AI isn't linear destination; it's cyclical refinement.
- **Mining adaptation:** Use for BCE pathway: Foundation (geology basics) → AI Understanding → Prompt Mastery → Hands-On Tools → Agent Design → Loop back to refine. Visual metaphor: pit optimization journey is circular (monitor → analyze → act → re-monitor).

### 2. **Stage Progression Visualizations** (Slide 41: Reactive → Single Agents → Agentic)
- **Pattern:** Three connected circles/stages with arrows, each showing increased capability. Comparison table below shows traits evolving across stages.
- **Why it works:** Clear mental model of capability maturity. Audiences understand "we are here" positioning.
- **Mining adaptation:** Apply to pit automation maturity: **Stage 1** = manual scheduling (operators plan shifts). **Stage 2** = AI scheduling agent (proposes shifts, operator approves). **Stage 3** = multi-agent pit orchestration (equipment + safety + logistics coordinate autonomously). Exact same visual template.

### 3. **Before/After Transformation Diagrams** (Slide 5: Current State → Target State)
- **Pattern:** Left box = "Ad-hoc AI Usage" (individual chats, manual processes, generic responses). Right box = "AI-Mature Organization" (integrated workflows, context-aware, strategic tools). Arrow between showing transformation.
- **Why it works:** Clarity on end-state value. Audiences see "what we're building toward."
- **Mining adaptation:** Current = "engineers using basic ChatGPT for pit questions." Target = "integrated system predicting ore grades, scheduling equipment, flagging safety issues, optimizing routes."

### 4. **Venn Diagram Overlaps** (Slide 31: Context Confusion)
- **Pattern:** Overlapping circles (e.g., "Early Incorrect Attempts" + "Current Contextual Information" = confusion zone). Annotations highlight failure mode.
- **Why it works:** Intuitive visualization of competing information causing degradation.
- **Mining adaptation:** Use for common pitfalls: "Old pit mapping data overlaps with new survey = conflicting AI recommendations." Venn diagram clarifies the problem visually.

### 5. **Layered Context Memory Models** (Slide 28-29)
- **Pattern:** Concentric rings or stacked boxes showing memory types (short-term, long-term, semantic, episodic, procedural, domain-specific). Each layer has role clearly labeled.
- **Why it works:** Audiences understand context isn't monolithic; it's structured by type & relevance.
- **Mining adaptation:** Apply to pit context: Innermost = current shift data (immediate relevance). Middle = equipment specs, historical logs (medium relevance). Outer = geological surveys, safety regulations (reference). Visual hierarchy aids comprehension.

### 6. **Real-World Example Progressions** (Slide 18: BPM → RPA → IPA Evolution)
- **Pattern:** Three parallel columns (Legal, Finance, HR), each showing 3-step evolution with before/after metrics (e.g., "15 steps → 1 week", "2 hours via RPA", "3 days via AI Intelligence"). Icons represent each step.
- **Why it works:** Concrete demonstration of value at each stage. Quantified gains = persuasive.
- **Mining adaptation:** Replace with 3 mining processes: **Equipment Maintenance** (process design → RPA scheduling → AI predictive), **Ore Grade Assessment** (manual → semi-automated → AI-optimized), **Safety Reporting** (paper forms → digital + RPA entry → AI anomaly detection). Keep structure; quantify gains per stage (e.g., "20 manual safety reports/day → 5 human review cases via AI filtering").

### 7. **Comparison Tables with Visual Cues** (Slide 29: Prompt vs Context Engineering)
- **Pattern:** 14-row table comparing aspects side-by-side. Color-coding or icons highlight which is superior for each dimension. Bottom banner emphasizes key insight ("Context is the DOUBLE OUTPUT").
- **Why it works:** Detailed comparison + clear winner per dimension. Removes ambiguity.
- **Mining adaptation:** Create table: "Manual Pit Scheduling vs. AI-Assisted Scheduling" with rows like "Time to Generate Schedule", "Adherence to Safety Rules", "Cost Optimization", "Flexibility to Changes", "Human Oversight Required". Each row shows manual vs. AI-assisted. Emphasize bottom banner: "AI doesn't replace engineering judgment; it amplifies it."

### 8. **Architecture Diagrams with Data Flow** (Slide 32: RAG, Slide 33: MCP)
- **Pattern:** Boxes (LLM, retriever, tools) connected by arrows showing data flow. Intermediate steps (routing, reranking) labeled. Color-coding distinguishes types (e.g., blue = retrieval, orange = generation).
- **Why it works:** Technical audiences understand system interactions. Less technical audiences see "multiple smart tools working together."
- **Mining adaptation:** Draw "Integrated Pit Monitoring System": Pit Sensors → Data Ingestion → Knowledge Store (geological, equipment data) → AI Agent (decision logic) → Equipment Controller (execution). Arrows show feedback loops (monitors → learns → improves).
